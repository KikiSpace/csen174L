# OpenAI & Hugging Face Classroom Demos

## 1. Project Name

**openai-photo-demo** — small, standalone Python scripts for teaching API usage (OpenAI vision, ElevenLabs TTS, Hugging Face inference). No web frontend.

## 2. Overview

This folder contains **independent terminal demos** for a CS classroom. Each script is meant to be run on its own. Dependencies are listed in `requirements.txt`. Secrets are loaded from a local `.env` file (see [Configuration](#7-configuration)).

The parent workspace (`lab1/`) currently holds only this project directory.

## 3. Features

| Demo | What it does |
|------|----------------|
| **Image description (OpenAI)** | Reads `photo.jpg`, sends it to the OpenAI Responses API with vision, prints a short text description. |
| **Image description + voice** | Same as above, then sends the description to ElevenLabs text-to-speech and saves `output.mp3`. |
| **Video transcription (Hugging Face)** | Extracts audio from `demo.mp4` with `ffmpeg`, transcribes via Hugging Face `InferenceClient`, prints the transcript and saves `transcript.txt`. |
| **Image classification (Hugging Face)** | Intended zero-shot classification of `photo.jpg` — **see warning below; not verified working.** |

### Experimental / broken example (read this)

**`classify_image_huggingface.py` is example-only material.** It was produced as AI-generated starter code, is **not** guaranteed to run, and **needs debugging** (for example, environment variable naming vs. documentation, API response shapes, and Hugging Face model availability may not match what the script expects). **Do not assume it works out of the box.** Use it only as a sketch for a future exercise or after you fix and test it locally.

Other Hugging Face–based scripts (`transcribe_video_huggingface.py`) may still fail depending on your Hugging Face account, model access, or inference routing; errors are printed to the terminal when that happens.

## 4. Repository Structure

```
lab1/

    ├── .env                 # Local secrets (create from .env_example; not committed if .gitignore applies)
    ├── .env_example         # Template for required environment variables

    ├── requirements.txt     # Python package dependencies

    ├── describe_image_with_voice.py         # Demo 2: OpenAI image → text → ElevenLabs → output.mp3

    ├── demo.mp4             # Sample video for transcription demo
```

**Note:** Large binaries (`demo.mp4`, `photo.jpg`, etc.) and generated files (`output.mp3`, `transcript.txt`, `demo_audio.wav`) are normal for local demos but may not belong in version control for every course; adjust `.gitignore` if you publish this repo.

## 5. Requirements

- **Python** 3.10+ recommended (3.13 is used in the included `venv`).
- **pip** and a virtual environment (recommended).
- **System:** [ffmpeg](https://ffmpeg.org/) on your `PATH` — required only for `transcribe_video_huggingface.py` (audio extraction from `demo.mp4`).

**Python packages** (see `requirements.txt`):

- `openai` — OpenAI API
- `python-dotenv` — load `.env`
- `elevenlabs` — ElevenLabs TTS (voice demo only)
- `huggingface_hub` — Hugging Face `InferenceClient` (transcription and classification demos)

## 6. Installation

From the `openai-photo-demo` directory:

```bash
cd openai-photo-demo
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

Install **ffmpeg** separately, e.g. on macOS with Homebrew:

```bash
brew install ffmpeg
```

## 7. Configuration

1. Copy the example env file and fill in your keys:

   ```bash
   cp .env_example .env
   ```

2. Edit `.env` with your real values. The template documents:

   - `OPENAI_API_KEY` — OpenAI (image demos)
   - `ELEVENLABS_API_KEY` — ElevenLabs (voice demo)
   - `HF_TOKEN` — Hugging Face (transcription demo)
   - `HF_TOKEN_READ` — referenced by the experimental `classify_image_huggingface.py` (may need alignment with `HF_TOKEN` when debugging)

Never commit `.env` or share API keys publicly.

## 8. How to Run

Run each script from **`openai-photo-demo`** so paths like `photo.jpg` and `demo.mp4` resolve correctly.

| Script | Prerequisites | Command |
|--------|----------------|---------|
| Image description | `OPENAI_API_KEY`, `photo.jpg` | `python describe_image.py` |
| Image + voice | `OPENAI_API_KEY`, `ELEVENLABS_API_KEY`, `photo.jpg` | `python describe_image_with_voice.py` |
| Video transcription | `HF_TOKEN`, `demo.mp4`, `ffmpeg` | `python transcribe_video_huggingface.py` |
| Image classification (experimental) | Hugging Face token per script / `.env`; `photo.jpg` | `python classify_image_huggingface.py` |

**Outputs:**

- `describe_image_with_voice.py` → `output.mp3`
- `transcribe_video_huggingface.py` → `demo_audio.wav` (intermediate), `transcript.txt`
